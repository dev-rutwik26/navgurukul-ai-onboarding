# 🏋️ Onboarding Exercises

Welcome to the hands-on part of your onboarding! This folder contains practical exercises that you will **complete, commit, and submit as Pull Requests** — just like real daily work.

---

## How This Works

1. Each exercise folder has a `task.py` (or similar) file with `# TODO:` markers
2. Read the instructions in each folder's `README.md`
3. Create your personal branch: `git checkout -b dev/your_name`
4. Complete the tasks, commit your work, and open a PR to `dev`
5. Your PR must pass code review before moving on

## Rules

- ✅ Complete exercises **on your personal branch** (`dev/your_name`)
- ✅ Each exercise = **one PR** with a linked Notion task
- ✅ All PRs need **at least 1 reviewer**
- ❌ Do not copy solutions without understanding them
- ❌ Do not commit AI-generated code you cannot explain line-by-line

---

> **Tip:** Treat each exercise like a production task. Write clean, modular, well-commented code.
